### 核心流程

```
main()
  │
  ├─ TcpListener::bind()
  ├─ server::run(listener, ctrl_c)
  │     │
  │     ├─ 初始化 Listener
  │     │     ├─ DbDropGuard::new()    → 启动后台过期清理任务
  │     │     ├─ broadcast::channel()  → notify_shutdown
  │     │     └─ mpsc::channel()       → shutdown_complete
  │     │
  │     ├─ tokio::select! ─────┬─ Listener.run()      ← 接受连接循环
  │     │                      └─ shutdown signal      ← ctrl_c
  │     │
  │     └─ 收到关闭信号后
  │           ├─ drop(notify_shutdown)     → 广播给所有 Handler
  │           ├─ drop(shutdown_complete_tx)→ 释放最后一个 Sender
  │           └─ shutdown_complete_rx.recv() → 等所有 Handler 结束
  │
  └─ 进程退出
```

### 要点

- **Listener** 是"工厂"——只负责创建 Handler，不管 Handler 死活
- **Handler** 是"工人"——每个连接一个，死循环 `read_frame → parse → apply → write_frame`
- **关闭信号**走两条通道：`broadcast` 一推多收通知退出，`mpsc` 计数确保所有工人收工
- **permit** 随 Handler 任务生，随 Handler 任务灭——自动归还信号量
